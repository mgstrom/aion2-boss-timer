# 上传到GitHub并制作EXE指南

## 文件清单

确保以下文件都已准备好：

### 核心文件（必须上传）
- ✅ `index.html` - 主页面
- ✅ `style.css` - 样式文件
- ✅ `script.js` - 逻辑文件
- ✅ `main.js` - Electron主进程
- ✅ `preload.js` - Electron预加载脚本
- ✅ `package.json` - 项目配置

### GitHub配置（必须上传）
- ✅ `.github/workflows/build.yml` - GitHub Actions工作流
- ✅ `.gitignore` - Git忽略文件
- ✅ `README.md` - 项目说明

## 上传步骤

### 1. 初始化Git仓库

在项目目录中打开命令提示符或PowerShell，执行：

```bash
# 初始化Git仓库
git init

# 添加所有文件
git add .

# 提交更改
git commit -m "Initial commit: Aion 2 BOSS Timer"
```

### 2. 创建GitHub仓库

1. 登录 [GitHub](https://github.com)
2. 点击右上角 "+" -> "New repository"
3. 输入仓库名称：`aion2-boss-timer`
4. 选择 "Public"（公开）或 "Private"（私有）
5. 不要勾选 "Initialize this repository with a README"（已存在README.md）
6. 点击 "Create repository"

### 3. 上传代码到GitHub

在GitHub仓库页面，复制以下命令（替换为你的用户名）：

```bash
# 添加远程仓库（替换YOUR_USERNAME为你的GitHub用户名）
git remote add origin https://github.com/YOUR_USERNAME/aion2-boss-timer.git

# 推送到GitHub
git branch -M main
git push -u origin main
```

### 4. 自动构建EXE

上传代码后，GitHub Actions会自动：
- 安装Node.js和依赖
- 构建便携版EXE：`永恒之塔2 BOSS刷新倒计时.exe`
- 构建安装版EXE：`永恒之塔2 BOSS刷新倒计时 Setup.exe`
- 上传构建产物

### 5. 下载EXE文件

**方法一：通过Artifacts下载**
1. 进入GitHub仓库
2. 点击顶部 "Actions" 标签
3. 选择最新的工作流运行（绿色✓表示成功）
4. 滚动到页面底部的 "Artifacts" 部分
5. 点击 "exe-files" 下载ZIP文件
6. 解压ZIP文件即可获得EXE文件

**方法二：创建Release（推荐）**
```bash
# 创建版本标签
git tag v1.0.0

# 推送标签到GitHub
git push origin v1.0.0
```

推送标签后，GitHub Actions会自动：
- 创建Release页面
- 上传EXE文件到Release

然后：
1. 进入GitHub仓库
2. 点击右侧 "Releases"
3. 找到最新版本
4. 下载EXE文件


## 注意事项

1. **不要上传 `node_modules` 文件夹** - 已在 `.gitignore` 中排除
2. **不要上传 `dist` 文件夹** - 构建输出，已在 `.gitignore` 中排除
3. **确保所有文件名正确** - 区分大小写
4. **GitHub Actions可能需要几分钟** - 请耐心等待构建完成

## 故障排除

### 构建失败
- 检查 `package.json` 格式是否正确
- 检查 `.github/workflows/build.yml` 是否存在
- 查看Actions日志获取详细错误信息

### 无法下载EXE
- 确保构建成功（绿色✓）
- 检查Artifacts是否已上传
- 如果是Release，确保标签已正确推送

## 功能说明

### BOSS倒计时
- 支持15个BOSS的刷新倒计时
- 可编辑BOSS剩余时间
- 支持提前声音提醒

### 小游戏闹钟
- 每小时第10分钟触发提醒
- 显示进场倒计时（到第18分钟）
- 可开关控制

### 数据存储
- 所有数据保存在浏览器localStorage中
- 刷新页面后数据不会丢失
